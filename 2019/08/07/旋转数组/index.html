旋转数组

#leetcode题目：旋转数组
题目描述：
给定一个数组，例如[1,2,3,4,5,6,7]，同时给定一个K值，表示数组的向右旋转程度，例如当k等于3时，旋转一次为：[7,1,2,3,4,5,6],旋转第二次为：[6,7,1,2,3,4,5],第三次也就是结果输出为：[5,6,7,1,2,3,4]。

##解法一：暴力解法
双重循环将数组内的元素进行移位：先将数组最左边的元素取出来，然后其他元素右移，再将取出来的元素放置在数组的第一个位置，循环K次即可。

class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        i = len(nums);
        t = i-2;
        while(k>0):
            temp = nums[i-1];
            while(t>=0):
                nums[t+1]=nums[t];
                t-=1;
            nums[0]=temp;
            t=i-2;
            k-=1;

时间复杂度：O(NK)
空间复杂度：O(1)

##解法二：利用数组存储
将数组中的后K位取出来，存储在另外一个数组中，再将新数组的元素逐个插入原数组的前面位置即可：

class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """     
        i=0
        tempNums=[];
        for each in nums[len(nums)-k:len(nums)]:
            tempNums.append(each);
            nums.pop();
        for each in tempNums:
            nums.insert(i,each);
            i+=1;

时间复杂度：O(N)
空间复杂度：O(N)

##解法三：将数组逆置
例如：1234567->7654321->5671234即可
时间复杂度：O(N)
空间复杂度：O(1)

##解法四：利用队列的性质
将数值的后K个(队尾)依次取出，再依次添加到队头

class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """   
        k=k%len(nums);
        i=0;
        while(i<len(nums)-k):
            nums.append(nums.pop(0));
            i+=1;

时间复杂度：O(N)
空间复杂度：O(1)

##解法五：环状替换
这是比较难理解的一个方法，首先，如果我们用之前的元素直接去替换它所应该最终在的位置上的元素（原位置+K%N就是它最终应该所在的位置），必然导致元素的丢失，所以，需要用一个变量(temp)先保存起来被替换的元素，继续这个过程N次即可，因为每个元素都要换位置，所以需要这个过程N次。但是有一个问题会出现，当N%K==0的时候，其中K=K%N(如果K>N,实际上还是相当于K%N,因为移动N+1个位置就相当于移动1个位置)循环替换多次后，一定会产生再次回到最初开始的位置上的问题，程序就进入死循环。此时，我们需要做的是将从一开始的元素的下一个元素开始继续这个过程。
详细的说就是一开始从i=0开始替换，并且保存每次被替换的元素，便于下次替换时候用上次被替换的元素去替换它应该替换的元素。i=0所在的元素替换i+k%N所在的元素，并且再次保存被替换的元素的值，此时替换的都是i%K==0的元素，并且因为每隔K替换一次，所以已经有N/K个元素确定位置了，当回到最初开始的元素时，就从开始的元素的下一个继续进行这个过程，此时就是i%k==1的元素在进行替换工作了，直到回到i%k==0的元素位置上，此时i==k,那么一共确定位置的就有k乘以N/K个元素了，刚刚好N个元素，所有位置都在指定的位置上。
过程如下，红线表示第一个阶段，i%k==0，绿线表示第二个阶段，i%k==1，两个阶段，每隔阶段(N=6/K=2)=3，2乘以3=6.所有元素归位

代码如下：

class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        count=0
        staIndex=0;#起点
        while(count<len(nums)):
            preValue=nums[staIndex];#预先值，即是将要放入确定位置的值
            curIndex=(staIndex+k)%len(nums);#当前可能需要被替换的值
            while(staIndex!=curIndex):#没有回到最初位置就继续循环
                temp=nums.pop(curIndex);#插入
                nums.insert(curIndex,preValue);
                preValue=temp;#更新数值
                count+=1;
                curIndex=(curIndex+k)%len(nums);
            nums.pop(curIndex);#插入本阶段最后数值后继续下一个阶段直到count=len（nums）即是所有数都放置在最终位置
            nums.insert(curIndex,preValue);
            count+=1;
            staIndex+=1;

时间复杂度：O(N)
空间复杂度：O(1)
